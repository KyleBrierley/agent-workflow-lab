# Sample Project Instructions

## Purpose

This fake project represents a small team repository. Keep changes scoped, explain assumptions, and validate the behavior you touch.

## Workflow

- Start by inspecting the relevant source, tests, and package scripts.
- Prefer a short plan before editing.
- Run `npm run validate` before finishing when the fixture includes a `package.json`.
- Keep final answers concise: changed files, validation, and one remaining risk.

## Config Boundary

Project instructions belong here. Execution settings such as sandboxing, approval policy, model choice, and web access belong in Codex config files.
